# Core Principles

## 1. Body-Only Output

微信公众号后台已经提供标题、作者、封面、摘要等字段。除非用户明确要求完整归档页，最终交付的 `article.wechat.html` 应只包含正文区域。

## 2. Text Formatting Is The Foundation

排版的基本盘是文字阅读体验：

- 段落长度
- 行距
- 段间距
- 标题层级
- 强调频率
- 引用与总结块
- 空白节奏

图片和组件不能破坏正文可读性。

## 3. Analyze Global Image Rhythm

图片不是孤立素材。要判断作者如何用图片控制整篇文章节奏：

- 每个板块是否都有图？
- 图片承担解释、转场、案例、总结，还是情绪铺垫？
- 图片与标题、图注、正文之间的位置关系是什么？
- 图片比例、宽度、背景、边框、留白是否稳定？

## 4. WeChat Compatibility First

HTML 在浏览器里好看，不等于能粘贴进公众号。默认应该使用更保守的结构：

- 简单标签
- 内联样式
- 少量 wrapper
- 不依赖脚本
- 不依赖复杂 CSS

## 5. Self-Check Every Run

每次生成都要检查：

- 是否误把标题/作者放进正文？
- 是否保留了原风格的阅读节奏？
- 是否有足够的图片和图注？
- 是否有网页端才能正常渲染的复杂组件？
- 是否提供了图片文件夹和上传对照表？
